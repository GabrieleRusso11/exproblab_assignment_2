State Machine Interface Module

This module implements an interaface used to handle the comunication between the state machine node and the other components of the architecture but it also contains functions used by the state machine node to compute data and to interact with the ontology through the Armor server

ROS Parameters:

/starting_map starting map file (it has the Ontology skeleton )

/operative_map is the file where the final map (and the complete ontology) will be loaded

/location_property property “hasDoor” associated to a location (room and corridor)

/robot_property property “isIn” to get and/or set the robot position in the map

/now_property property “now” is a robot time property

/urgent_property property “urgencyThreshold” to change the urgency threshold, associated to the locations

/visited_property property “visitedAt” used to change the timestamp of a location, when the robot visits it

/canreach_property property “canReach” to get the locations that the robot can reach

/coord_x_property property “Coordinate_X” to get and/or set the X coordinate of the location

/coord_y_property property “Coordinate_Y” to get and/or set the Y coordinate of the location

/robot_name the name that identify the robot

/charging_location the name of the location where is the charging station

Subscriber:

/state/battery_low receives from the robot state node the robot battery level

/marker_publisher/detected_marker_id receives from the marker_publish node of the aruco_ros package, the id of the marker detected by the camera

Publisher:

/robot/joint1_position_controller/command sends to the joint 1 (the base of the manipulator) position controller, the position commands. In order to rotate the base of the manipulator

/robot/joint2_position_controller/command sends to the joint 2 (the prismatic joint associated to the link of the manipulator) position controller, the position commands. In order to move down the link where is attached the camera

Service:

/inteface/start_charging sends to the robot state node, the value of the start charging flag, as service request, in order to handle the robot charging process .

/room_info takes from the marker_server node, the location informations associated to the markers detected

Action Client:

/motion/controller sends to the controller node the path plan as goal, and receives the final location reached as result.

/move_base used to cancel the move base goal, when the battery level becomes low.
